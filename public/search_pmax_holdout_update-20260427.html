Facebook API outage flag · 4/20 – 4/23, backfilled 4/24

What happened, and what it did to the test

Facebook user account restriction took the FB purchase event API down for ~3 days. Last-click on Facebook fell off a cliff and the FB algorithm likely degraded. We added the clean window (3/14–4/19) to anchor against the prior run, and the extended window (3/14–4/26) to capture the freshest read.

Counter-intuitive finding: the FB outage week helped the S&P incrementality read, not hurt it. Possible reasons — (1) FB backfill weakened during the outage, widening the lit-vs-dark gap, (2) AOV gap was widest that week (lit $228 vs dark $198), (3) both. Either way, the clean window is the cleaner read; the extended is the most current. Recommendation lands the same way under either.

What's next

Recommendations

  • Keep S&P at current nationwide spend. Test confirms net-positive on revenue and roughly cost-neutral after backfill.
  • Branded vs non-branded Search holdout (Q2 2026). Search appears to carry more weight than PMAX in regression. If branded Search is dragging the blend down, isolating it would let us reduce that spend.
  • PMAX-only holdout in parallel. PMAX LC GPM (+$64k delta) and Stella PMAX benchmark (2.98x) suggest PMAX is a strong solo channel — clean isolation would resolve whether the blended number is mostly Search or mostly PMAX.
  • DMA-level regression for AOV-protection hypothesis. Descriptive evidence is consistent (lit AOV always above dark); weekly OLS isn't decisive at n=14. A DMA × week panel would have hundreds of observations.
  • Re-test in Q4 2026 in a clean window. No mid-test promos, no API outages. Each run has shifted iROAS upward; a clean rerun would tighten the estimate.
